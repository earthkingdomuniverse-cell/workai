# WorkAI Growth Loops

## Purpose

Growth loops for WorkAI 1.0. Each loop connects product behavior to measurable growth, liquidity, retention, or trust.

## 1. Supply growth loop

Provider completes successful deal -> provider invites more providers -> more offers -> more matching opportunities.

### Trigger

Provider completes a deal successfully.

### Action

- Share/invite prompt after completed deal.
- Referral link or invite code.
- Suggested invite copy based on provider category.

### Reward

- More relevant work opportunities.
- Reputation/trust benefit or future referral incentive.

### KPI

- Invite-to-signup rate
- Signup-to-offer rate
- Providers per referral

## 2. Demand growth loop

Requester gets useful outcome -> shares WorkAI -> more requesters -> more demand.

### Trigger

Requester completes a deal successfully.

### Action

- Share result prompt.
- Invite similar founders/builders/operators.
- Suggested request template for next need.

### KPI

- Shares per completed deal
- Referral signup rate
- Referral signup to first request rate

## 3. Proposal/deal conversion loop

More relevant proposals -> more accepted proposals -> more deals -> more reputation -> better proposal conversion.

### Trigger

Proposal is accepted or rejected.

### Action

- AI improves matching and proposal quality suggestions.
- Provider gets feedback on proposal quality.
- Requester gets next best action to accept, reject, or request clarification.

### KPI

- Proposal-to-deal rate
- Time to accept
- Proposal response rate

## 4. Review/trust loop

Completed deals -> reviews -> trust score improves -> better visibility -> more deals.

### Trigger

Deal reaches released/completed state.

### Action

- Prompt both parties to leave review.
- Display trust score impact.
- Feed trust score into ranking and match confidence.

### KPI

- Review completion rate
- Average trust score change
- Deal conversion by trust tier

## 5. AI utility loop

Users use AI -> AI suggestions lead to actions -> outcomes train better recommendations -> more AI usage.

### Trigger

User uses AI match, price, support, or next-action.

### Action

- Track suggestion click-through.
- Ask lightweight feedback where useful.
- Use accepted/rejected outcomes as learning signals.

### KPI

- AI usage rate
- AI suggestion click-through rate
- AI-assisted conversion lift

## 6. Operator safety loop

Risk/dispute signal -> operator resolves issue -> user trust preserved -> marketplace remains healthy.

### Trigger

Dispute, fraud signal, or high-risk deal appears.

### Action

- Operator review queue prioritizes high-risk items.
- Operator records decision.
- Outcome feeds back into risk/trust rules.

### KPI

- Time to resolution
- Retention after issue
- Repeat issue rate
- Dispute rate by category

## Priority order

| Loop | Priority | Reason |
| --- | --- | --- |
| AI Utility | High | Core differentiation |
| Proposal/deal conversion | High | Marketplace survival |
| Review/trust | High | Trust and conversion |
| Operator safety | High | Retention and risk control |
| Supply | Medium | Growth |
| Demand | Medium | Growth |

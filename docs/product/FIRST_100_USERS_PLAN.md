# WorkAI First 100 Users Plan

## Purpose

Practical plan for sourcing, qualifying, onboarding, activating, and retaining the first 100 WorkAI users.

## Target mix

| Metric | Target |
| --- | --- |
| Total signups | 100 |
| Providers | 70 |
| Requesters | 30 |
| Profile complete | 80 |
| First action | 50 |

## Sourcing users

### Priority 1: Personal network

| Source | Target | Type |
| --- | --- | --- |
| Friends who freelance | 15 | Provider |
| Friends who need freelancers | 10 | Requester |
| Alumni / coworkers | 10 | Mixed |

### Priority 2: Warm outreach

| Source | Target | Type |
| --- | --- | --- |
| Design/freelance groups | 15 | Provider |
| Startup founders | 10 | Requester |
| Dev/design communities | 10 | Mixed |

### Priority 3: Referral

| Source | Target | Type |
| --- | --- | --- |
| Early user referrals | 15 | Both |
| Warm introductions | 10 | Both |
| Pre-collected email list | 5 | Both |

## Who to invite first

- Active freelancers with portfolio
- Startup founders with real needs
- Agency owners with overflow work
- Consultants with clear skills

## Who not to invite early

- Competitor staff
- Purely curious users
- Users with no time to test
- Users with no budget or no supply value
- Trolls or bad actors

## Activation milestones

| Milestone | Target timing |
| --- | --- |
| Signup | Immediate |
| Onboarding complete | Within 48 hours |
| First offer/request | Within 7 days |
| First proposal | Within 14 days |
| First deal | Within 30 days |

## Provider activation path

1. Signup
2. Complete onboarding
3. Create offer
4. Receive or find matched requests
5. Send proposal
6. Convert to deal

## Requester activation path

1. Signup
2. Complete onboarding
3. Create request
4. Receive proposals
5. Accept proposal
6. Convert to deal

## Retention plan

| Day | Action |
| --- | --- |
| D0 | Welcome and first action tips |
| D1 | Check-in nudge |
| D3 | Personal follow-up if no action |
| D7 | Feedback request |
| D14 | Feature or opportunity update |
| D30 | Referral ask |

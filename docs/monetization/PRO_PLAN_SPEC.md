# WorkAI Pro Plan Spec

## Purpose

Prepare possible WorkAI Pro monetization for 1.x while protecting 1.0 growth and liquidity.

## Best-fit paying users

- High-volume providers
- Agencies
- Frequent requesters
- Operators or teams with repeated work needs

## Free core

These should remain free at 1.0 and early 1.x:

- Create profile
- Create offer/request
- Submit proposal
- Complete first deal
- Leave reviews
- Basic trust profile
- Basic messaging when available

## Candidate Pro features

| Feature | Value | Gate type |
| --- | --- | --- |
| Analytics dashboard | High | Soft |
| Priority visibility | High | Soft and clearly labeled |
| Custom badge | Medium | Soft |
| Multiple profiles or team profiles | Medium | Hard later |
| API access | High | Hard later |
| Advanced AI copilot | High | Soft/usage-based |

## What not to gate early

- First deal
- Onboarding
- Basic profile
- Viewing offers/requests
- Basic proposals
- Reviews
- Trust score visibility

## Growth risks

| Risk | Reason |
| --- | --- |
| Paywall too early | Kills liquidity before marketplace proves value |
| Gating wrong features | Users churn before activation |
| Paid visibility without labeling | Damages trust |
| Paid trust signals | Breaks marketplace integrity |

## Recommended timing

WorkAI Pro should wait until:

- users repeat behavior,
- deal flow is proven,
- trust score matters in conversion,
- and analytics/AI features create clear incremental value.

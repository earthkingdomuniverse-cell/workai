# WorkAI Documentation

This folder contains product, architecture, launch, QA, monetization, investor, analytics, and operating documents for WorkAI.

## Structure

- `architecture/` - product and technical architecture
- `product/` - product scope, user journeys, activation, retention, growth, liquidity, notifications, and referrals
- `launch/` - go-live, launch scope, communication, staging, support/incident ops, experiments, and post-launch plans
- `marketing/` - app store and landing page copy
- `monetization/` - pricing, fee, revenue, pro plan, enterprise monetization
- `investor/` - investor summaries, fundraising narrative, pitch materials, and business-plan docs
- `analytics/` - event tracking and analytics dashboard specifications
- `qa/` - QA inventory, final QA summary, regression and release quality reports
- `reports/` - build reports, bug reports, seed reports, project status reports

## Root files kept intentionally

- `README.md` stays at repository root as the main entrypoint.
- `WORKAI_REBRAND_REPORT.md` currently stays at root while the rebrand is still in progress.

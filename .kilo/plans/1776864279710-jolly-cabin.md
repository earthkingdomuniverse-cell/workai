# Payments QA Plan

## Scope

- Payments/Transactions/Receipts module
- Backend API
- Mobile screens
- Field mapping between backend and mobile
- Pseudo-payment (mock) data consistency

## QA Tasks

### 1. Backend API Tests

#### 1.1 Transactions List

- Test: `GET /api/v1/transactions`
- Verify: returns list with all fields
- Expected fields: id, dealId, amount, currency, type, status, referenceNumber, userId, userName, createdAt

#### 1.2 Transaction Detail

- Test: `GET /api/v1/transactions/{id}`
- Verify: returns exact transaction

#### 1.3 Receipts by Deal

- Test: `GET /api/v1/deals/{id}/receipts`
- Verify: returns receipts for specific deal

#### 1.4 Receipt Detail

- Test: `GET /api/v1/receipts/{id}`
- Verify: returns exact receipt

#### 1.5 Filter by dealId

- Test: `GET /api/v1/transactions?dealId=xxx`
- Verify: filtering works

#### 1.6 Filter by type (fund/release/refund/payout)

- Test: `GET /api/v1/transactions?type=fund`
- Verify: type filtering works

#### 1.7 Filter by status

- Test: `GET /api/v1/transactions?status=completed`
- Verify: status filtering works

### 2. Field Mapping Tests

  ```
  'fund' | 'release' | 'refund' | 'payout' | 'service_fee' | 'dispute_resolution'
  ```
- Mobile (mobile/src/services/paymentService.ts:8): SAME
- Status: CHECK

#### 2.2 TransactionStatus Enum

- Backend (src/types/transaction.ts:4):
  ```
  'pending' | 'processing' | 'completed' | 'failed' | 'cancelled' | 'refunded'
  ```
- Mobile (mobile/src/services/paymentService.ts:9): SAME
- Status: CHECK

#### 2.3 Receipt Status

- Backend: 'paid' | 'refunded' | 'void'
- Mobile: same
- Status: CHECK

#### 2.4 All Receipt Fields

- Compare: id, dealId, transactionId, amount, currency, items, subtotal, serviceFee, total, paidAt, receiptNumber, status, pdfUrl, createdAt, updatedAt

### 3. Mobile UI Tests

#### 3.1 Payment Screen (payment.tsx)

- Check: dealId passed correctly
- Check: amount displayed correctly
- Check: form validation (card fields)
- Check: amount formatting (USD symbol)

#### 3.2 Receipts Screen (receipts.tsx)

- Check: receipts load from dealId
- Check: receipt display renders
- Check: service fee shown
- Check: total calculation (subtotal + serviceFee)

#### 3.3 ReceiptCard Component

- Check: exists
- Check: fields rendered correctly

### 4. Data Consistency Tests

#### 4.1 Amount Formatting

- 5000 should display as "$5,000" not "5000"
- Currency symbol present

#### 4.2 Date Formatting

- ISO dates rendered in readable format

#### 4.3 Service Fee Calculation

- Check: serviceFee = amount _ 5% (from mock data: 5000 _ 5% = 250)
- Check: total = subtotal + serviceFee

#### 4.4 Type Mapping

- 'fund' = money from client to escrow
- 'release' = money from escrow to provider
- 'refund' = money back to client
- 'payout' = provider withdrawal
- 'service_fee' = platform fee

### 5. Edge Cases

#### 5.1 Non-existent transaction ID

- Expected: 404 error

#### 5.2 Non-existent receipt ID

- Expected: 404 error

#### 5.3 Deal with no receipts

- Expected: empty array

#### 5.4 Invalid type filter

- Expected: ignored or 400

### 6. Known Issues to Check

#### 6.1 Missing backend routes

- Check if GET /transactions supports query params (currently returns all)
- Check if GET /receipts supports dealId filter (currently not implemented in routes.ts lines 74-75)

## Output

- qa/payments-checklist.md with test cases and results
- Bug reports for any mismatches found
